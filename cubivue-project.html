A cloud based ‘Software as a Service’ (SaaS) solution to improve processes of business

CubiVue is a ‘Software as a Service’ (SaaS) solution, designed to improve business processes in many industries. This cloud based setup can be used in just couple of hours. The solution is designed to ensure higher and optimized efficiency in order handling, reducing logistic errors, and can expand the performance efficiency. I led the mobile app design system across all the industries and optimized the exisitng user experience of the app.

    Problem

    The intial app was serving for a single warehouse industry but later on we expanded support for major industries as well which required us to create a unified design system for the app which was called as CubiVue App. It will support all the basic required features by major industries. Originally, there was no generalized app for the future industries, where our UX resources could map new industries on the exisiting platform to create wireframes and mockups for the customers.

    • Solution
    • I built a simplified design system for the mobile application to be used by UX resources for their customers and for the app developers who may utilize this unified design components and features for the base app. The end product helps digitize the processes of understanding and creating a standard for the app which are easily demostratable for the user and for the developers to create base app.
    • Approach
    • I worked in a team with a fellow designer and a product manager to design the end-to-end UI/UX for the Android and iOS application. It was used for designing the base app and mapping future industries to serve optmized experience for their specified processess. I worked closely with my team to gather design requirements, feedback, conduct user testing on wireframes, functional prototypes and visual designs, both for Android and iOS. I also worked with a team of developers to ensure delivery of the final product.

    Who is this for?

    User Research was critical to this project since we were designing for diverse type of industries which were unknown to us. One of the first things we did was, to draft a basic information architecture to understand the common key values. We identified various common variables which we can serve to our clients in the base app. We gathered information from different industries where we were already serving our customers e.g. Cleaning industry etc.

    Findings

    We identified the common variables we required for the base CubiVue app. By researching and consulting with the stakeholders, we streamlined our basic features which we will give in our base app.

    What does the mockup look like?

    Upon gathering features layout and identifying required information, I carefully designed the mockups for the base CubiVue app. This design system will help future resources to map new industries and also it will give technical design guide for the developers so they can all collaborate effeciently.

    Telling a story with data

    One of the biggest challenge for designing this platform was, to map all the required information which will help UX resources to use it to present it in a meaningful format for the upcoming industries. I wanted the UI and flows to achieve the following objectives:

    • Add basic flows and features to the base CubiVue app
    • Design a detailed system which defines all components and the guides for the developers and stakeholders
    • Bird's eye view of all the industry's flows
    • Future support for upcoming industries to map it on the base CubiVue app
